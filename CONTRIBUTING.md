# Как контрибьютить в MyLibrary

## Процесс

1. Форкни репозиторий
2. Создай ветку: `git checkout -b feature/название`
3. Сделай изменения и напиши тесты
4. Убедись что тесты проходят: `dotnet test`
5. Закоммить: `git commit -m "feat: описание"`
6. Запушь: `git push origin feature/название`
7. Открой Pull Request в ветку `master`

## Правила

- Каждый PR должен содержать тесты
- Коммиты в формате: `feat:`, `fix:`, `docs:`, `ci:`
- Один PR — одна задача
- CI должен быть зелёным перед merge

## Локальная разработка
```bash
dotnet build
dotnet test
```
